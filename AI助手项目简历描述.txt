汽车配件仓库管理系统 — AI 智能助手
全栈开发 / AI 应用开发
2026.01 - 2026.06

为仓库管理系统开发 AI 智能助手，支持用自然语言查询库存、统计数据、导航页面、检索操作指南。
核心方案是 Prompt-Based Function Calling：通过 System Prompt 定义 8 个可用工具（查询库存、
低库存预警、统计数据、知识库检索等），Qwen3-Max 大模型理解用户语义后返回 JSON 调用指令，
后端解析后调用真实业务接口获取数据，解决传统关键词匹配（"火花塞还有多少" 被错误提取为
"火花塞还"）的边界问题。

- 设计 Prompt-Based Function Calling 架构，替代 120 行 if-else 关键词匹配 + JSON 意图识别代码，
  消除正则提取配件名的边界情况，模型自行决定调用哪个工具和参数

- 基于 LangChain4j 1.0.0 封装阿里云百炼 DashScope OpenAI 兼容接口，统一 ChatModel、
  StreamingChatModel、EmbeddingModel 等 5 个 Bean 的创建和配置

- 实现 RAG 向量检索流水线：DocumentByParagraphSplitter(500字) 对 5 份知识库文档分段 →
  text-embedding-v3 分批嵌入(每批8个) → InMemoryEmbeddingStore 存储 →
  用户提问时余弦相似度搜索(TopK=3, minScore=0.3) → 拼接增强 Prompt → LLM 生成回答

- 实现 SSE 流式推送：后端完成工具调用后将回复文本自适应分块（标点单独断句，中文2-4字/块，
  20-50ms间隔），通过 Server-Sent Events 推送 token/done 事件，前端 EventSource 接收
  实现打字机效果

- 集成多轮对话记忆（InMemoryChatMemoryStore，每会话最多20条消息），对话历史拼接后传入
  Prompt 供 LLM 理解上下文

- 添加滑动窗口速率限制（20次/分钟/会话，ConcurrentHashMap 实现）和审计日志
  （内存500条缓冲，记录请求/响应/token/耗时）

- 打通后端 7 个 Service（StockService、PartService、InboundService、OutboundService 等），
  构建从用户提问 → LLM 语义理解 → 工具调度 → 业务接口调用 → 结果返回的全自动闭环

技术栈：Vue 2.7、Element UI、Spring Boot 2.7、MyBatis-Plus、LangChain4j 1.0.0、
阿里云百炼 Qwen3-Max / text-embedding-v3、SSE、RAG
