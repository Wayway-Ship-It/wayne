# AI 智能助手 — 需求文档

## 概述

为汽车配件仓库管理系统开发一个 AI 智能助手，用户通过自然语言与系统交互，实现库存查询、页面导航、操作指引、数据统计等功能。

## 功能需求

### FR1: 自然语言库存查询
- **描述**: 用户输入配件名称 + "库存"/"数量"，返回该配件的当前库存、安全库存、单位
- **示例**: "查一下前车门库存" → 返回前车门库存信息
- **优先级**: P0

### FR2: 低库存预警查询
- **描述**: 用户询问库存不足的配件列表
- **示例**: "哪些配件库存不足" → 返回低于安全库存的配件清单
- **优先级**: P0

### FR3: 页面智能导航
- **描述**: 用户说出目标功能模块，自动跳转到对应页面
- **示例**: "打开入库单页面" → 跳转到 /inbound
- **优先级**: P1

### FR4: 操作指引
- **描述**: 用户询问操作步骤，返回基于知识库的操作指引
- **示例**: "怎么新增供应商" → 返回供应商新增步骤
- **优先级**: P1
- **增强**: 使用 RAG 从知识库检索相关文档

### FR5: 数据统计查询
- **描述**: 用户询问统计数据，返回聚合结果
- **示例**: "本月入库总数量" → 返回入库统计
- **优先级**: P2

### FR6: 流式对话输出
- **描述**: AI 回复通过 SSE 实时逐字输出，提升响应体验
- **优先级**: P2

### FR7: 多轮对话记忆
- **描述**: 支持上下文理解的多轮对话
- **示例**: "查刹车片库存" → "那入库单呢" → 理解上下文跳转到入库页面
- **优先级**: P3

### FR8: AI 用量统计
- **描述**: 管理员可查看 AI 调用次数、Token 消耗
- **优先级**: P3

## 非功能需求

### NFR1: 安全性
- API Key 不得硬编码在代码中
- 用户输入需做安全过滤

### NFR2: 性能
- 本地规则匹配响应 < 50ms
- LLM 调用超时 30s
- 流式首 Token 延迟 < 2s

### NFR3: 可用性
- LLM 调用失败时提供友好降级提示
- 支持重试机制

### NFR4: 可维护性
- 知识库文档与代码分离
- 意图识别规则可配置
- 开发日志每日记录
