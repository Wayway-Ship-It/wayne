## 2️⃣ 数据库初始化（关键步骤！）

### 2.1 为什么需要初始化数据库？

**登录失败的最主要原因是数据库没有初始化！**

- 数据库中没有用户表
- 没有默认的管理员账户
- 没有示例数据

### 2.2 使用Navicat初始化数据库（推荐）

#### 步骤1：打开Navicat for MySQL
- 双击桌面上的Navicat图标
- 或从开始菜单打开

#### 步骤2：连接到本地MySQL
1. 点击"连接" → "MySQL"
2. 填写连接信息：
   - 连接名：`本地MySQL`（或任意名称）
   - 主机：`localhost`
   - 端口：`3306`
   - 用户名：`root`
   - 密码：`xmx992992`
3. 点击"测试连接"，确保连接成功
4. 点击"确定"保存连接

#### 步骤3：选择数据库
- 在左侧连接列表中，双击刚创建的连接
- 找到并点击 `auto_parts_warehouse` 数据库
  - 如果没有这个数据库，先右键连接名 → "新建数据库"
  - 数据库名：`auto_parts_warehouse`
  - 字符集：`utf8mb4`
  - 排序规则：`utf8mb4_unicode_ci`

#### 步骤4：执行初始化脚本
1. 点击顶部菜单的"查询" → "新建查询"
2. 在查询窗口中，点击"文件" → "打开SQL文件"
3. 浏览并选择文件：`d:\TRAE\index\database\init.sql`
4. 点击查询窗口上方的"运行"按钮（▶️ 图标），或按 `F5` 键
5. 等待执行完成，下方会显示"成功"消息

#### 步骤5：验证数据初始化
执行以下查询验证数据：
```sql
USE auto_parts_warehouse;
SELECT * FROM sys_user;
```

应该能看到2条用户记录（admin和user）。

### 2.3 使用命令行初始化数据库（备选）

如果Navicat不可用，可以使用命令行：

```powershell
# 1. 配置MySQL路径
$mysqlPath = "C:\Program Files\MySQL\MySQL Server 8.0\bin"
$env:Path = "$mysqlPath;$env:Path"

# 2. 执行初始化脚本
mysql -u root -pxmx992992 < d:\TRAE\index\database\init.sql
```

## 3️⃣ 启动后端服务

### 3.1 方式一：使用批处理脚本（推荐）

1. 打开文件夹：`d:\TRAE\index\`
2. 双击文件：`启动后端.bat`
3. 等待启动完成（约30-60秒）
4. 看到类似以下日志表示启动成功：
   ```
   Tomcat started on port(s): 8080 (http)
   Started WarehouseApplication in X.XXX seconds
   ```

### 3.2 方式二：使用命令行启动

```powershell
# 1. 配置Maven环境
$env:MAVEN_HOME = "D:\Apache\apache-maven-3.9.14\apache-maven-3.9.14-bin\apache-maven-3.9.14"
$env:Path = "$env:MAVEN_HOME\bin;$env:Path"

# 2. 进入后端目录
cd d:\TRAE\index\auto-parts-warehouse-backend

# 3. 启动后端
mvn spring-boot:run
```

### 3.3 验证后端服务

打开浏览器访问：http://localhost:8080

如果看到错误页面（Whitelabel Error Page）是正常的，说明后端已启动。

## 6️⃣ 常见问题排查

### 问题1：登录时提示"服务不可用"

**可能原因：**
- 后端服务没有启动
- 后端端口被占用

**解决方法：**
1. 检查后端是否在运行
2. 确认端口8080没有被其他程序占用
3. 重新启动后端服务

### 问题2：登录时提示"用户名或密码错误"

**可能原因：**
- 数据库没有初始化
- 输入的用户名或密码错误

**解决方法：**
1. 确认已执行数据库初始化脚本（第2步）
2. 检查Navicat中sys_user表是否有数据
3. 使用默认账户：admin / admin123

### 问题3：前端页面无法打开

**可能原因：**
- 前端服务没有启动
- 端口被占用

**解决方法：**
1. 检查前端是否在运行
2. 查看终端显示的实际端口号
3. 重新启动前端服务

### 问题4：Maven命令找不到

**可能原因：**
- Maven环境变量未配置

**解决方法：**
```powershell
# 临时配置Maven环境
$env:MAVEN_HOME = "D:\Apache\apache-maven-3.9.14\apache-maven-3.9.14-bin\apache-maven-3.9.14"
$env:Path = "$env:MAVEN_HOME\bin;$env:Path"

# 验证
mvn -version
```

### 问题5：MySQL连接失败

**可能原因：**
- MySQL服务没有启动
- 密码错误

**解决方法：**
1. 确认MySQL服务正在运行
2. 确认密码是：xmx992992
3. 使用Navicat测试连接
