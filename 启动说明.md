# 汽车配件仓库管理系统 - 启动说明

## 常见问题排查

### 问题1：服务不可用

如果遇到"服务不可用"的问题，请按照以下步骤排查：

#### 第一步：检查环境要求

确保已安装以下环境：
- **JDK 1.8 或更高版本**
- **MySQL 8.0 或更高版本**
- **Maven 3.6 或更高版本**（用于后端编译）
- **Node.js 14 或更高版本**（用于前端）

检查命令：
```bash
java -version
mvn -version
node -v
mysql --version
```

#### 第二步：数据库配置

1. 启动MySQL服务
2. 创建数据库并执行初始化脚本：
```sql
-- 使用Navicat或命令行执行 database/init.sql
```

3. 修改后端配置文件 `auto-parts-warehouse-backend/src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/auto_parts_warehouse?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false
    username: root      # 改为你的MySQL用户名
    password: your_password  # 改为你的MySQL密码
```

#### 第三步：启动后端服务

方法1：使用Maven（推荐）
```bash
cd auto-parts-warehouse-backend
mvn clean install
mvn spring-boot:run
```

方法2：使用IDEA
- 用IntelliJ IDEA打开 `auto-parts-warehouse-backend` 目录
- 等待Maven依赖下载完成
- 运行 `WarehouseApplication.java` 主类

后端启动成功后，访问 http://localhost:8080 测试

#### 第四步：启动前端服务

```bash
cd auto-parts-warehouse-frontend
npm install
npm run serve
```

前端启动成功后，访问 http://localhost:8081

#### 第五步：登录测试

使用以下默认账户登录：
- **管理员**：用户名 `admin`，密码 `admin123`
- **普通用户**：用户名 `user`，密码 `admin123`

## 详细错误排查

### 后端启动失败

**错误1：数据库连接失败**
```
Communications link failure
```
解决方法：
1. 确认MySQL服务已启动
2. 检查数据库地址、端口、用户名、密码是否正确
3. 确认数据库 `auto_parts_warehouse` 已创建

**错误2：端口被占用**
```
Port 8080 was already in use
```
解决方法：
1. 修改 `application.yml` 中的端口号
2. 或者关闭占用8080端口的程序

**错误3：Maven依赖下载失败**
解决方法：
1. 配置Maven镜像源（阿里云镜像）
2. 检查网络连接

### 前端启动失败

**错误1：npm install 失败**
解决方法：
1. 使用淘宝镜像：`npm install --registry=https://registry.npmmirror.com`
2. 删除 `node_modules` 文件夹和 `package-lock.json` 后重新安装

**错误2：端口被占用**
解决方法：
1. 前端默认使用8081端口，如被占用会自动使用其他端口
2. 查看终端输出的实际访问地址
